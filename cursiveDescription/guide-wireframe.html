Design note: A Guide is like an application template. It contains a sequence of pages (prompts/exercises). When a facilitator "runs" this guide in a group, it generates worksheet instances for participants. The guide itself is reusable—you can use it across multiple groups, fork it, or publish it to a library.
The flow:
Guide (template) ↓ Facilitator uses in Group ↓ "Send next page" → creates mail_item ↓ Participant clicks "Begin" → creates worksheet section ↓ Worksheet lives in group folder (not in guide) ↓ Submission creates snapshot → goes to Review ↓ If curated → goes to Gallery (with consent)
White-label implications:
  • Domain independence: A guide URL could be cursive.ai/guides/phil101 OR school.edu/guides/phil101
  • Custom vocabulary: Each instance can rename primitives (exercise→reflection, gallery→anthology)
  • Forking: Anyone can fork a published guide, modify it, and run their own version
  • Cross-instance: A guide published on cursive.ai could be imported/used on school.edu
Values alignment:
  • Human curation: The guide is human-authored, not generated
  • Renamability: Vocabulary respects different contexts (classroom vs salon vs newsroom)
  • Slowness: Pages are released over time, not all at once
  • Transparency: Students see who created the guide, can see the template
