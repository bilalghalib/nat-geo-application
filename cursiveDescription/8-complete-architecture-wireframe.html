Cursive: Convivial Infrastructure for Embodied Learning

A federated protocol for human correspondence, dialogue, and collaborative making

Core Primitives

Layer 1: Content (Portable)
Letter
1+ pages of writing/drawing. Self-contained, exportable as JSON/Markdown. Can be private, public, or shared via magic link.
Guide
Sequence of letters + schedule + facilitation settings. Importable/exportable across instances. Has vocabulary, license, attribution.
Layer 2: Identity (Federated)
Identity
alex@cursive.com or bilal@westwood.edu. Works across all instances. ActivityPub compatible. Has public profile, RSS feed.
Instance
cursive.com, westwood.edu, etc. Self-hostable, federated. Instances can share guides, subscribe to feeds, discover users.
Layer 3: Organization (Groups)
Group
A cohort using a guide. Has facilitators (owner + co-facilitators) and participants. Lives on one instance but can use guides from others.
Mail
Delivery system for letters. Can be in-app notifications, actual email, or both. Supports scheduling, magic links, postponing.
Layer 4: Context (Knowledge Graph)
Knowledge Graph
Entities (people, concepts, places) extracted from letters. Opt-in, privacy-preserving. Enables discovery and connections.
Whispers
AI suggestions that appear in chrome (not in content). Connection suggestions, related concepts, embodied activities.
